# 任务名称
实现合规层基础能力（Compliance Layer）

## 任务目标
把产品中的合规要求系统化，避免开发过程中因为文案或展示方式不当而偏离定位。
本任务不追求复杂风控，而是先建立“基础合规护栏”。

## 产品背景
本产品是一个：
- 数学模拟工具
- 行为复盘工具
- 决策影响展示工具

本产品不是：
- 投顾系统
- 交易建议系统
- 价格预测系统

因此系统中的关键页面和生成内容必须明确体现：
“这是数学模拟，不是金融建议。”

## 本任务要完成的内容
### 1. 定义统一的合规文案常量
请建立统一常量，例如：
- 全局简短声明
- 页面底部数学说明
- 分享报告强制水印

必须包含以下强制文案：
[SYSTEM] This is a mathematical simulation. Not a financial recommendation.

### 2. 实现统一的合规展示组件
建议实现以下组件：
- ComplianceNotice
- FormulaExplanationNotice
- ReportWatermark

这些组件后续会被以下页面复用：
- sandbox 页面
- shadow 页面
- report 页面

### 3. 实现基础敏感措辞过滤能力
请实现一个简单的文本检查工具函数，用于检查 AI 或动态文案中是否出现高风险措辞。

至少拦截或标记以下表达：
- 建议买入
- 建议卖出
- 抄底
- 必涨
- 必跌
- 底部确认
- will go up
- will go down
- strong buy
- strong sell
- financial advice

可以输出：
- 是否命中
- 命中的词
- 建议替换为中性措辞

### 4. 给出页面接入示例
请至少提供在以下页面中如何使用这些组件的示例：
- sandbox 页面底部
- report 页面底部

## 技术要求
- 使用 TypeScript
- 代码放在 `lib/compliance` 和 `components` 等合适目录
- 工具函数与 UI 组件分开
- 便于后续扩展更复杂的文案过滤逻辑

## 输出结果
你需要输出：
1. 合规常量
2. 合规组件
3. 文本敏感措辞检查工具
4. 使用示例

## 验收标准
- 关键文案统一管理
- 强制水印可复用
- 文本检查工具可正常识别敏感措辞
- 组件可以直接在页面中接入
- 不使用复杂外部服务
- 不过度设计

## 编码要求
- 结构清晰
- 命名明确
- 注释简洁
- 保持 MVP 可用优先
