# 任务名称
实现加仓成本模拟器页面与实时交互（Sandbox Form & Realtime UI）

## 任务目标
实现产品第一核心页面：加仓成本模拟器。
用户输入当前持仓信息和模拟加仓条件后，页面应实时展示新的平均成本、成本改善效率和回本阈值。
这是用户停留时间最长的页面，必须做到交互清晰、响应快、风格统一。

## 依赖前置
本任务依赖以下基础能力已经存在：
- 项目基础骨架
- 像素风基础组件
- 数学引擎
- 合规展示组件

请尽量复用已有模块，不要重复造轮子。

## 产品背景
这个页面要帮助用户快速回答：
1. 我现在继续加仓后，平均成本会变成多少？
2. 我花这笔钱，成本改善值不值？
3. 现在价格要反弹多少，我才能回本？
4. 不同加仓方案，哪个更有效？

页面必须强调：
- 这是数学模拟
- 不是买卖建议
- 不预测未来走势

## 本任务要完成的内容
### 1. 实现 Sandbox 页面
建议页面路由：
- `/sandbox`

### 2. 实现输入区域
至少包含以下输入项：
- 当前持仓股数
- 当前平均成本
- 当前市价

模拟输入支持两种方式：
- 加仓金额
- 加仓股数

两种方式至少支持其一。
如果你设计得合理，也可以支持二选一切换。

### 3. 实现实时结果展示
当输入变化时，实时展示以下结果：
- 新平均成本
- 成本改善效率
- 回本阈值（百分比）

结果要清晰分组展示，并具有像素风 UI 风格。

### 4. 实现多方案对比（MVP 版）
支持暂存最多 3 个模拟方案。
每个方案至少显示：
- 加仓金额或加仓股数
- 新平均成本
- 成本改善效率
- 回本阈值

展示方式可选：
- 卡片对比
- 简单柱状图
- 或二者结合

注意：
MVP 阶段不必一开始就强行接数据库。
如果你认为先用前端本地状态实现更稳，也可以先做本地版，但代码结构要便于后续接入保存接口。

### 5. 接入合规说明
在页面底部接入：
- 数学模拟说明
- 非金融建议声明

### 6. 做好状态处理
至少处理：
- 输入为空
- 输入非法
- 数值为 0 或负数
- 无法计算时的提示

## 技术要求
- 使用 TypeScript
- 使用已有 Pixel UI 组件
- 调用已有数学引擎函数
- 状态管理尽量简单清晰
- 页面应适合桌面端优先，兼顾基础响应式

## 输出结果
你需要输出：
1. `/sandbox` 页面
2. 输入表单组件
3. 结果展示组件
4. 多方案对比区域
5. 基础状态处理
6. 合规说明接入

## 验收标准
- 页面可正常打开
- 输入变化后结果实时刷新
- 结果数值来自数学引擎，而不是页面中重复计算
- 支持最多 3 个方案对比
- 页面风格与像素设计系统一致
- 页面底部有合规声明
- 不出现“建议买入/卖出”等措辞

## 编码要求
- 优先保证可用性和清晰性
- 不要把大量计算逻辑写死在页面组件里
- 尽量把表单、结果展示、方案卡片拆成子组件
- 注释使用简洁英文
- 不要过度设计复杂状态机
