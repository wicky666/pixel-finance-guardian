# AI 成本模拟器 · 项目全览

> 用通俗的话说清楚：这个项目是什么、能干什么、怎么跑起来。

---

## 一、项目是干啥的？

**一句话：** 帮你在动手加仓/减仓之前，先算清楚「动了会怎样」，顺便提醒你「别乱动」。

**不是：** 不预测涨跌、不给买卖建议、不接券商下单。

**是：** 一个数学模拟 + 行为复盘工具，让你看清楚自己的操作习惯，减少冲动交易。

---

## 二、你能在界面上干哪些事？

| 入口 | 功能 |
|------|------|
| **先别急着动**（首页） | 输入持仓（代码、股数、成本、现价）→ 选加仓/减仓/不动 → 输入金额或股数 → 看「新成本」「回本价」「风险变化」→ 点「保留这次模拟」 |
| **你最近怎么动的** | 看你的操作风格（冷静 vs 冲动）、最容易乱动的时候、最值得反思的一点 |
| **如果你当时没动** | 对比「实际做了」和「如果没做」的差异，用一句话提醒你 |
| **我的复盘** | 每天生成一段人格化复盘（四段式），可切换标准/分享风格，一键导出图片发小红书/抖音 |

---

## 三、项目是怎么拆的？（三端）

借鉴你 GitHub 上 aiforce 的思路：**用户端、管理端、后端** 分目录管理。

```
仓库根目录
├── app/、components/...     → 用户端 Web（你看的主界面）
├── apps/api/               → 后端 API（存数据、算东西）
├── apps/admin/             → 管理端（内部看记录，MVP 只读）
└── packages/config、shared → 共享配置和类型
```

- **不接后端时**：数据全在浏览器本地（localStorage），关掉就没了，但功能照样用。
- **接后端时**：在 `.env.local` 里配 `NEXT_PUBLIC_API_BASE_URL=http://localhost:3100`，保存模拟会同步到 Nest，管理端能看到。

## 五、数据流（简化版）

1. 你在首页填持仓、点「保留这次模拟」  
   → 本地立刻保存 + 如果配了后端，再发一份到 API。

2. 「你最近怎么动的」「如果你当时没动」  
   → 读本地或 API 里的模拟记录，算出行为/影子对照。

3. 「我的复盘」  
   → 根据记录生成四段文字，可按风格/人格切换，导出图片。

4. 管理端  
   → 调 API 看模拟列表和影子对照（当前只读）。

## 七、常见问题

**Q：数据会丢吗？**  
不接后端时，关浏览器/清缓存会丢。接后端后，数据在 API 内存里，重启 API 会丢；正式上线会用数据库，就不会丢。

**Q：行情从哪来？**  
A 股用腾讯兜底，其他用 Yahoo。国内网络下 A 股更稳。

**Q：管理端账号密码？**  
MVP 阶段没有登录，直接打开就能看，后续再接鉴权。
