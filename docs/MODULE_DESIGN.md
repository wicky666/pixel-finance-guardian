# 模块设计与链路闭环（参考：盈鑑概覽 + 行為模式分析）

## 一、设计意图解读

参考两张图的设计意图，产品应具备：

1. **概覽（首頁）** — 类似「盈鑑·概覽」
   - **核心指標**：總回報、勝率（勝/負）、盈虧比、期望盈虧、日誌完整度、平均持倉；可選篩選（全部/港股/美股/A股，本項目可簡化為「全部」或按標的）。
   - **圖表區**：累計盈虧曲線（可切換盈虧 / 滾動勝率）、入場情緒 × 勝率（冷靜/興奮/焦慮/FOMO）。
   - **近期快覽**：近期模擬/交易快覽表（市場、股票、入場價、入場理由、盈虧、AI點評摘要）；無數據時引導至「成本模擬」或「交易日誌」。

2. **行為分析（行為模式分析）** — 專題頁
   - **頭部**：標題「行為模式分析」、日期範圍選擇（年/月/日）、按鈕「生成完整報告」。
   - **三圖**：情緒分佈、入場理由勝率、持倉天數 × 盈虧（散點，盈利/虧損分色）。
   - **六大行為指標**：冷靜 vs 情緒化勝率、有計劃 vs 衝動交易勝率、止損執行率、多重信號確認(3+)勝率等；每項可帶進度條或對比文案。

3. **報告與導出**
   - 支持按日期範圍「生成完整報告」，報告內容覆蓋核心指標 + 行為指標 + 免責聲明，可截圖/打印導出。

本項目定位為「模擬 + 行為復盤」，不替代真實交易系統，因此：
- 「總回報/勝率/盈虧比」等可基於**模擬方案與 Real/Shadow 快照**衍生（例如模擬盈虧、模擬勝率）。
- 「交易日誌」可預留為擴展（記錄「執行了的加倉」與「未執行的對比」），MVP 先以「已保存方案」當作「近期快覽」數據源。
- 「入場情緒」在無實盤數據時可由「模擬頻次、間隔、理智值」推導為簡化標籤（如冷靜/衝動），用於圖表占位與閉環。

## 二、模塊劃分（與導航對齊）

| 模塊 | 路由 | 對應參考圖 | 職責 |
|------|------|------------|------|
| **概覽** | `/` | 盈鑑概覽 | 首頁儀表盤：核心指標卡片、累計盈虧曲線（或影響曲線）、入場情緒×勝率圖、近期方案快覽表；無數據時引導至成本模擬。 |
| **成本模擬** | `/sandbox` | — | 輸入持倉與加倉，實時結果，保存方案（最多 3），生成 Real/Shadow 快照。 |
| **影子賬戶** | `/shadow` | — | Real vs Shadow 對比、當前影響、趨勢條、歷史快照列表。 |
| **行為分析** | `/behavior` | 行為模式分析 | 日期範圍、生成完整報告按鈕；情緒分佈、入場理由勝率、持倉×盈虧三圖（可占位）；六大行為指標列表（理智值、計劃vs衝動、止損執行率等）。 |
| **報告** | `/report` | 兩圖均有「報告」能力 | 日期範圍、生成完整報告；摘要（標的、影響、理智值、指標匯總）、強制水印與免責；支持截圖/打印導出。 |

擴展預留（不納入當前導航）：
- **交易日誌**：記錄「實際執行」的加倉與結果，與模擬對齊，供勝率/盈虧比計算。
- **數據管理**：導入/導出、清空本地數據。

## 三、數據閉環（主鏈路）

```
[用戶輸入持倉] → [成本模擬] → [保存方案] → [生成 Real/Shadow 快照]
       ↓
[概覽] ← 讀取方案+快照：核心指標、累計影響曲線、情緒標籤、近期快覽表
       ↓
[影子賬戶] ← 讀取快照：Real vs Shadow、Impact、歷史
       ↓
[行為分析] ← 讀取方案+快照+行為服務：日期範圍內 情緒分佈、入場理由勝率、持倉×盈虧、六大指標
       ↓
[報告] ← 日期範圍 + 生成完整報告：摘要 + 水印 + 導出
```

說明：
- **成本模擬** 是唯一寫入「方案 + Real/Shadow」的入口；刪除方案時同步刪除對應快照。
- **概覽** 是首屏，聚合「核心指標 + 兩圖 + 近期快覽」，數據來自 store + shadowService + behaviorService。
- **行為分析** 在現有「理智值 + 24h 活動」基礎上，擴展為「日期範圍 + 六大行為指標 + 三圖占位」，並提供「生成完整報告」跳轉或彈窗。
- **報告** 支持選擇日期範圍，組裝「完整報告」數據並展示，帶水印與導出提示。

## 五、後端/服務擴展要點

| 服務 | 現狀 | 擴展方向（對齊參考圖） |
|------|------|--------------------------|
| store | 方案 + Real/Shadow 快照 | 不變；可加「標的類型」篩選（若做市場篩選）。 |
| shadowService | Impact、歷史、圖表數據 | 累計影響曲線序列（按時間）。 |
| behaviorService | 24h 次數、間隔、是否顯著 | 日期範圍內統計；入場理由/情緒標籤（由模擬類型、頻次推導）。 |
| sanityService | 理智值、等級、原因 | 六大行為指標中「冷靜vs情緒化」「計劃vs衝動」可基於現有原因與次數推導。 |
| reportService | 摘要、水印、生成時間 | 支持日期範圍參數；輸出「完整報告」結構（含六大指標摘要）。 |
| **新增/擴展** | — | **行為指標服務**：止損執行率（可占位 0/0）、多重信號確認勝率（可占位）；情緒分佈、入場理由勝率、持倉×盈虧的圖表數據結構（可先占位或簡化）。 |

## 六、實施順序建議

1. **文檔與導航**：已按本設計更新模塊與閉環；導航改為「概覽、成本模擬、影子賬戶、行為分析、報告」。
2. **概覽重構**：首頁由「簡短介紹+按鈕」改為儀表盤：核心指標 6 卡、累計曲線圖占位、情緒×勝率圖占位、近期方案快覽表。
3. **行為分析深化**：日期範圍、生成完整報告按鈕、六大行為指標列表（含止損執行率、多重信號勝率等占位）、三圖占位。
4. **報告**：日期範圍選擇、生成完整報告（帶日期參數）、六大指標摘要。
5. **服務層**：擴展 reportService/getReportData 支持 dateFrom/dateTo；新增或擴展「六大指標」計算（可先占位返回 0 或靜態結構）。

以上完成後，主鏈路閉環為：**成本模擬 → 保存方案 → Real/Shadow → 概覽看總覽 → 行為分析看指標與圖 → 報告生成與導出**。
