# 本地测试指南

按下面步骤在本地把项目跑起来，并简单自测一遍。

---

## 第一步：装依赖

在项目根目录执行：

```bash
npm install
```

---

## 第二步：跑起来（两种方式）

### 方式 A：只跑用户端（最简单）

```bash
npm run dev
```

浏览器打开：**http://localhost:3000**。

- 不配后端时，数据只在浏览器本地，功能都可用。
- 若 3000 被占用，Next 会自动改用 3001 或 3002，终端会提示实际地址。

### 方式 B：用户端 + 后端 + 管理端（完整联动）

开 **三个终端**，依次执行：

| 终端 | 命令 | 地址 |
|------|------|------|
| 1 | `npm run api:dev` | API: http://localhost:3100 |
| 2 | `npm run dev` | 用户端: http://localhost:3000 |
| 3 | `npm run admin:dev` | 管理端: http://localhost:5173 |

要验证「保存同步到后端」时，需先配置环境变量：

1. 复制 `.env.example` 为 `.env.local`
2. 设置：`NEXT_PUBLIC_API_BASE_URL=http://localhost:3100`
3. 重启 `npm run dev`

---

## 第三步：功能自测清单

按下面顺序点一遍，确认都能用：

- [ ] **首页**：输入股票代码（如 600519）→ 点「获取最新价」→ 当前价自动填入
- [ ] **首页**：填持仓（股数、成本、现价）→ 选加仓 → 输入金额 50000 → 点「试着算一下」→ 看到新成本、回本价
- [ ] **首页**：点「保留这次模拟」→ 出现成功提示
- [ ] **你最近怎么动的**：能看到行为分析（若还没记录，可能提示「记录还不够多」）
- [ ] **如果你当时没动**：能看到实际 vs 没动的对比（需先有模拟记录）
- [ ] **我的复盘**：点「生成复盘」→ 看到四段内容 → 点「切换风格」→ 点「生成图片」→ 下载成功
- [ ] **管理端**（若开了）：http://localhost:5173 → 能看到健康检查、模拟列表、影子对照

---

## 第四步：跑单元测试

```bash
npm run test
```

正常会看到类似：

```
✓ lib/compliance/sensitive-wording.test.ts
✓ lib/math/sandbox.test.ts
Test Files  2 passed (2)
Tests  29 passed (29)
```

---

## 第五步：构建检查

```bash
npm run build
npm run api:build
npm run admin:build
```

三个都成功即表示可正常打出生产包。

---

## 遇到问题？

| 现象 | 处理 |
|------|------|
| 白屏或 500 | 执行 `npm run clean` 清缓存，再 `npm run dev` |
| 「获取最新价」无反应 | 检查网络；A 股 600519 一般可用，海外需能访问 Yahoo |
| 管理端空白 / 报错 | 确保 API 已启动（`npm run api:dev`），管理端通过代理访问 `/api` |
| 端口被占用 | 修改 `.env.local` 的 `API_PORT` 或关掉占用端口的程序 |

---

## 快速命令汇总

| 命令 | 说明 |
|------|------|
| `npm run dev` | 启动用户端 |
| `npm run api:dev` | 启动后端 |
| `npm run admin:dev` | 启动管理端 |
| `npm run test` | 跑单测 |
| `npm run build` | 构建用户端 |
| `npm run clean` | 清 Next 缓存（出问题时用） |
